# Code of Conduct

## Our commitment

We want Anduril to be a welcoming, respectful, and constructive project for everyone.

Participants in this project agree to contribute to a community that is harassment-free, inclusive, and focused on collaboration.

## Expected behavior

Examples of behavior that help create a positive environment include:

- being respectful and professional
- assuming good intent while giving clear feedback
- accepting constructive criticism gracefully
- focusing discussion on the work, not the person
- sharing context and rationale when disagreeing
- helping newer contributors learn the project

## Unacceptable behavior

Examples of unacceptable behavior include:

- harassment, intimidation, or discrimination
- personal attacks, insults, or trolling
- publishing private information without permission
- sexualized language or unwelcome sexual attention
- sustained disruptive behavior in issues, reviews, or discussions
- knowingly making the project less safe for contributors or users

## Scope

This Code of Conduct applies to all project spaces, including:

- GitHub issues
- pull requests and code review
- documentation discussions
- chat and other official community channels

## Enforcement responsibilities

Project maintainers are responsible for clarifying and enforcing this Code of Conduct and may remove, edit, or reject comments, commits, code, issues, and other contributions that do not align with it.

## Reporting

If you experience or witness unacceptable behavior:

- do not escalate publicly
- contact the maintainers through a private GitHub channel when possible
- if the report is security-sensitive, follow `SECURITY.md`

Maintainers will review reports promptly and handle them as confidentially as practical.

## Enforcement guidelines

Possible maintainer responses include:

- private warning
- temporary moderation of participation
- removal of content
- temporary or permanent ban from project spaces
