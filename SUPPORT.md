# Support

## Where to ask for help

Use GitHub issues for:

- bug reports
- feature requests
- documentation problems
- setup problems that appear reproducible

Please include:

- what you tried
- expected behavior
- actual behavior
- logs or screenshots when helpful
- environment details such as OS, .NET SDK version, provider/integration used, and whether Docker was involved

## Before opening an issue

- read `README.md`
- check the relevant setup guide under `docs/`
- search existing issues first

## Security issues

Do **not** report vulnerabilities in public issues.

Follow `SECURITY.md` instead.

## Scope of support

Anduril is maintained as an open-source project and support is best-effort.

Responses may be slower for:

- provider-specific edge cases
- third-party service outages or API changes
- experimental or lightly used integrations

## Contributions welcome

If you already understand the root cause, a pull request with tests and documentation is often the fastest path to resolution.
